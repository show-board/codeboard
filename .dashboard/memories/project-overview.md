# CodeBoard 项目功能及目的介绍

## 项目名称
CodeBoard — VibeCoding 多项目看板

## 项目目的
为 AI Agent（Cursor、Claude Code、OpenClaw 等）驱动的 VibeCoding 开发模式提供一个统一的任务管理和状态追踪看板。解决多项目并行开发时任务状态不可视、项目记忆丢失、Agent 协作无法同步的问题。

## 核心功能

### 1. 多项目看板
- 横向滚动的项目列视图，每列一个项目
- 支持项目颜色标识、隐藏、丢弃、恢复
- 实时消息色块提示各项目最新动态

### 2. Session 与任务追踪
- 每个项目下按时间排列 Session 卡片
- 每个 Session 包含目标、任务列表、状态
- Session 完成后显示任务总结，支持查看详细时间线
- 「期待卡片」机制：白条带子交互，输入提示词后可复制发送给 Agent

### 3. 记忆系统
- 9 类标准化项目记忆（基础记忆 1-6 + 运行记忆 7-9）
- Agent 通过 API 读写记忆，实现跨 Session 知识传递
- 记忆管理界面支持在线查看和编辑

### 4. Agent 对接
- 提供标准 REST API（端口 2585）
- 提供 SKILL.md 指导文件，Agent 按流程自动对接
- 支持 Socket.IO 实时推送
- 支持系统通知（macOS 原生通知）

### 5. 辅助功能
- 用户头像自定义（本地图片选择 + 1:1 裁剪）
- Skills 模板生成器（根据当前 IP/端口自动生成）
- API 文档全屏查看
- 使用帮助指南

## 目标用户
使用 AI Agent 进行日常编码的开发者，特别是同时管理多个项目的 VibeCoding 实践者。
